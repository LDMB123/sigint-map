## Documentation Standards

### File Naming
- **Uppercase with underscores**: AGENT_TEMPLATE.md
- **Descriptive names**: Clear purpose from filename
- **No dates in names**: Use git history for versioning

### Content Standards
- **Markdown format**: GitHub-flavored markdown
- **Code examples**: Always include working examples
- **Version info**: Document when things change
- **Cross-links**: Link to related documentation
