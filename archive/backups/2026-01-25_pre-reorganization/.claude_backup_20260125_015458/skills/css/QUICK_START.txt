## Code Example: Before & After

### Pattern: Hover State (5 minutes to fix)

**BEFORE (JavaScript + CSS)**
```tsx
const [isHovered, setIsHovered] = useState(false);

return (
  <Card
    onMouseEnter={() => setIsHovered(true)}
    onMouseLeave={() => setIsHovered(false)}
    style={{ transform: isHovered ? 'translateY(-4px)' : 'translateY(0)' }}
  />
);
```

**AFTER (Pure CSS)**
```css
.card {
  transition: transform 0.2s ease;
}

.card:hover {
  transform: translateY(-4px);
}
```

**Benefit:** 40 lines removed, eliminate state updates, smoother animation

## Pro Tips

### Tip 1: Start with Low-Risk Skills
`css-nesting.md` and `logical-properties.md` are lowest risk.
Move to `scroll-driven-animations.md` next.
Save `apple-silicon-optimization.md` for when performance matters.

### Tip 2: Test Early and Often
After each skill, run:
- Visual regression test
- Cross-browser test (Chrome, Firefox, Safari)
- Performance profile with DevTools
- Mobile testing (iPad, Android)

### Tip 3: Implement Incrementally
Don't do all 5 skills at once. Choose a path:
- **Path A (Quick):** css-nesting → apple-silicon-optimization
- **Path B (Scroll-heavy):** scroll-driven-animations → apple-silicon-optimization
- **Path C (Full):** All 5 in sequence over 4-6 weeks

### Tip 4: Measure Everything
Each skill includes "Expected Output" with metrics.
Record before/after numbers for:
- Bundle size

- Core Web Vitals

### Tip 5: Involve Your Team

- Weeks 3+: Implement with pair programming

## Common Questions

**Q: Can I do just one skill?**
A: Yes! Each skill is independent. Start with whichever addresses your biggest pain point.

**Q: Do I need all 5 skills?**
A: No. Mix and match based on needs:
- Performance focused? → apple-silicon-optimization + scroll-driven-animations
- Sass removal? → Just css-nesting
- International? → Just logical-properties
- Full modernization? → All 5

**Q: Will this break my site?**
A: No. All skills include @supports fallbacks and browser compatibility info. Gradual rollout recommended.

**Q: How long does implementation take?**
A: 1-2 weeks solo, 2-4 weeks with proper team process, including review and testing.

**Q: What about Safari users?**
A: All skills work on Safari 15+. Some advanced features (scroll animations) have fallbacks.
