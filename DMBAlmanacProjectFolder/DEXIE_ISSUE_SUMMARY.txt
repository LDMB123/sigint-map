HIGH PRIORITY ISSUES
================================================================================

ISSUE 8: Compression Monitor Metrics Not Exported
  File: /src/lib/db/dexie/data-loader.ts:343-366, 1317
  Problem: compressionMonitor.recordLoad() called but module may throw
  Risk: Silent failures if monitor crashes
  Fix: Wrap monitor calls in try-catch

ISSUE 9: bulkOperation Mode Always 'rw'
  File: /src/lib/db/dexie/query-helpers.ts:560-625
  Problem: Hardcoded 'rw' mode causes unnecessary write locks
  Risk: Read-only operations block concurrent writes
  Fix: Add transactionMode parameter (default 'rw' for writes)

ISSUE 10: Cache Cleanup Never Initialized
  File: /src/lib/db/dexie/cache.ts:664-687
  File: /src/lib/db/dexie/init.ts (missing call)
  Problem: startPeriodicCacheCleanup() defined but never called
  Risk: Cache grows unbounded, memory leak
  Fix: Call setupCacheInvalidationListeners() + startPeriodicCacheCleanup() in init

ISSUE 11: Missing [songId+showDate] Index Usage
  File: /src/lib/db/dexie/queries.ts:527-542
  Problem: getShowsForSong uses songId only, ignores compound index
  Risk: 30-50% slower than necessary (from schema.ts line 660)
  Fix: Use .where('[songId+showDate]').between(...) instead

ISSUE 12: Sync Module No Transaction Semantics
  File: /src/lib/db/dexie/sync.ts (lines 1+)
  Problem: Server→Client sync updates not atomic
  Risk: Partial updates if operation fails mid-sync
  Fix: Wrap all sync operations in transactions

ISSUE 13: No Batch Retry Logic
  File: /src/lib/db/dexie/query-helpers.ts:560-625
  Problem: bulkOperation fails on first error, no retry
  Risk: Transient IndexedDB locks cause entire operation to fail
  Fix: Add retry loop with exponential backoff

ISSUE 14: Cache TTL Not Validated
  File: /src/lib/db/dexie/cache.ts:336-350
  Problem: Cache TTL values chosen arbitrarily, no bounds checking
  Risk: TTL too short wastes queries, too long stales data
  Fix: Add MIN_TTL/MAX_TTL validation, make environment-aware

ISSUE 15: No Query Performance Metrics
  File: /src/lib/db/dexie/queries.ts + query-helpers.ts
  Problem: No visibility into slow queries
  Risk: Can't detect regression when indexes degrade
  Fix: Add timedQuery() helper with performance thresholds
