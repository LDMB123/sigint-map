CSS ANCHOR SYNTAX
-----------------

Define Anchor:
  <div style:anchor-name="--my-anchor">Content</div>

Position Element:
  <div style:position-anchor="--my-anchor" class="positioned">
    Content
  </div>

CSS Rules:
  .positioned {
    position: absolute;
    position-anchor: var(--position-anchor);
    inset-area: bottom;           /* or: top, left, right */
    margin-top: 8px;              /* Distance from anchor */
    position-try-fallbacks: flip-block;  /* Auto-flip */
  }

CSS PROPERTIES REFERENCE
------------------------

anchor-name: --my-anchor
  → Identifies anchor point

position-anchor: var(--position-anchor)
  → Associates with anchor

inset-area: bottom
  → Positions below anchor (also: top, left, right)

anchor-size(width)
  → References anchor's width

position-try-fallbacks: flip-block
  → Auto-repositions if doesn't fit

POSITION OPTIONS

TOOLTIP:
  position="top"     → Above trigger, flips to bottom
  position="bottom"  → Below trigger, flips to top
  position="left"    → Left of trigger, flips to right
  position="right"   → Right of trigger, flips to left

DROPDOWN:
  position="top"     → Above trigger
  position="bottom"  → Below trigger (default)

POPOVER:
  position="top"     → Above trigger
  position="bottom"  → Below trigger (default)
  position="left"    → Left of trigger
  position="right"   → Right of trigger

QUICK TROUBLESHOOT
-------------------

Positioning wrong?
  → Check position-anchor value
  → Verify anchor-name on trigger
  → Check inset-area value

Width not matching trigger?
  → Use min-width: anchor-size(width)
  → Check @supports blocks

Keyboard not working?
  → Check role="menu" / role="menuitem"
  → Verify focus management code
  → Test with actual keyboard

Mobile positioning off?
  → Test on real device (not zoom)
  → Check viewport meta tag
  → Verify touch target size (44x44px)
