INCREMENTAL SYNC
================

First sync: Full fetch (limited by SYNC_LIMITS)
Subsequent: Fetch only data updated since last sync
- Downloads: 100-500 KB vs 2-5 MB for full
- Speed: ~500ms to 2s vs 5-30s for full
- Network savings: 80%+ reduction
- Battery: ~1% vs 5% for initial

======================

GET /api/guests (if available)

All must support pagination:
- page: integer
- limit: integer
- sortBy: string
- sortOrder: 'asc' | 'desc'
- Optional: since (timestamp for incremental)

Response format (required):
{
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 50,
    "total": 1000,
    "hasMore": true
  }
}
