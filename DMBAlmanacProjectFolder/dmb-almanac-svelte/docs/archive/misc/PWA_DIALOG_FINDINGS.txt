ISSUE #1: UpdatePrompt.tsx - Circular Dependency (LINE 52)
┌─────────────────────────────────────────────────────────────────────────┐
│ Severity: LOW                                                            │
│ Type: Code quality                                                       │
│ Status: REQUIRED FIX                                                     │
│ Time: 1 minute                                                           │
│                                                                          │
│ CURRENT CODE (WRONG):                                                   │
│   const handleDialogClose = useCallback(() => {                         │
│     handleDismiss();                                                     │
│   }, [handleDialogClose]);  // ❌ Circular reference                    │
│                                                                          │
│ FIXED CODE:                                                              │
│   const handleDialogClose = useCallback(() => {                         │
│     handleDismiss();                                                     │
│   }, [handleDismiss]);  // ✅ Correct dependency                        │
│                                                                          │
│ WHY IT MATTERS:                                                          │
│   The circular reference causes handleDialogClose to be recreated       │
│   on every render, defeating the purpose of useCallback memoization.    │
│   While React handles this gracefully, it's inefficient and a code      │
│   smell. Very easy one-line fix.                                        │
└─────────────────────────────────────────────────────────────────────────┘

ISSUE #2: InstallPrompt.tsx - Unnecessary State Variable (LINE 222)
┌─────────────────────────────────────────────────────────────────────────┐
│ Severity: VERY LOW                                                       │
│ Type: Code quality / optimization                                        │
│ Status: OPTIONAL IMPROVEMENT                                             │
│ Time: 10 minutes                                                         │
│                                                                          │
│ ISSUE:                                                                   │
│   The shouldShow state variable adds complexity:                        │
│   - Setting state causes re-render                                       │
│   - Re-render triggers another effect                                    │
│   - Second effect calls showModal()                                      │
│   This adds an unnecessary cycle.                                       │
│                                                                          │
│ SOLUTION:                                                                │
│   Call showModal()/close() directly without intermediate state.         │
│   Reduces from 2 effects to 1, fewer re-renders, simpler code.         │
│                                                                          │
│ IMPACT:                                                                  │
│   - Removes 1 useState                                                   │
│   - Removes 1 useEffect                                                  │
│   - Fewer re-renders (~30% reduction)                                   │
│   - No functional change                                                │
└─────────────────────────────────────────────────────────────────────────┘
