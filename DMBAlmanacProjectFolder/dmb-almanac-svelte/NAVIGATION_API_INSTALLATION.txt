BASIC USAGE EXAMPLES
================================================================================

Navigate to a page:
  import { navigateWithTransition } from '$utils/navigationApi';
  await navigateWithTransition('/shows');

Show back button:
  import { navigationStore, canGoBack } from '$stores/navigation';
  <button disabled={!$canGoBack} onclick={() => navigationStore.goBack()}>
    Back
  </button>

Navigate with analytics:
  import { navigateWithTracking } from '$utils/navigationApi';
  await navigateWithTracking('/songs', {
    trackingInfo: { source: 'search' }
  });

View history:
  import { historyEntries } from '$stores/navigation';
  {#each $historyEntries as entry}
    <button onclick={() => navigationStore.goTo(entry.url)}>
      {entry.url}
    </button>
  {/each}

BROWSER SUPPORT
================================================================================

Chrome 143+ (recommended)     - Full support with all features
Chrome 102-142               - Full support (no Chrome 143+ features)
Chrome < 102                 - Fallback to History API (automatic)
Edge 102+                    - Full support (same as Chrome)
Safari 18+                   - View Transitions only
Firefox                      - Fallback to History API (automatic)

Key Point: No breaking changes! Automatic fallback for older browsers.

DEBUGGING
================================================================================

Check if Navigation API available:
  window.navigation  // Should return Navigation object

Enable debug logging:
  import { enableNavigationLogging } from '$lib/hooks/navigationApiInterception';
  enableNavigationLogging();

Access debug info:
  window.__navigationContext.getHistory()      // All navigations
  window.__navigationContext.getStats()        // Statistics
  window.__navigationContext.getLast()         // Most recent
  window.__navigationContext.isNavigating()    // Current state

Monitor store:
  import { navigationStore } from '$stores/navigation';
  navigationStore.subscribe(state => console.log(state));

Check View Transitions:
  document.activeViewTransition  // Current transition (Chrome 143+)

FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Do I need to do anything to use this?
A: No! It's automatically initialized in +layout.svelte.

Q: Does it work on older Chrome?
A: Yes! Automatic fallback to History API for Chrome < 102.

Q: Will it work on other browsers?
A: Yes! Safari works with View Transitions. Firefox falls back gracefully.

Q: Do I need any external dependencies?
A: No! Pure TypeScript/Svelte implementation.

Q: How do I debug navigation?
A: Use window.__navigationContext or enable debug logging.

Q: Can I customize View Transitions?
A: Yes! CSS @view-transition rules can be customized per route.

Q: How do I add analytics?
A: Use navigateWithTracking() or configure analytics hook.

Q: Is this production-ready?
A: Yes! Fully tested and documented.
