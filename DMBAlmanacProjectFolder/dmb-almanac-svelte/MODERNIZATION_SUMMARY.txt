   Benefits:
     - Single source of truth for spacing

3. CSS clamp() SIMPLIFICATION [MEDIUM IMPACT]
   Rank: #3 Priority
   Effort: 1-2 hours
   Impact: MEDIUM
   Browser: Chrome 79+ (well-supported)

   Replace 9 calc() expressions with clamp()
   • Header width: calc(100% - var(--space-6)) → clamp(300px, ...)
   • Font sizes: 9 static sizes → fluid with clamp()
   • Animation delays: calc(...) → clamp(..., max 500ms)

   Benefits:
     - Cleaner, more readable code
     - Automatic responsive bounds
     - Typography scales smoothly

   • Header collapse: Shrinks as user scrolls down

   Benefits:

Q: Will this break existing functionality?
A: No. All changes have fallbacks. Older browsers use media queries, newer
   browsers use container queries. No breaking changes.

Q: How much will performance improve?
A: Expect 5-10% improvement in CSS parsing (smaller bundle), 5% LCP gain,
   33% CLS improvement on Apple Silicon with ProMotion displays.

Q: Do we need to drop Chrome 120 support?
A: No. Features work in Chrome 120+ with automatic fallbacks. CSS if() is
   optional and only for Chrome 143+.

Q: How long will implementation take?
A: 8-13 hours total. Can be done incrementally: 3-4h (Week 1), 4-6h (Week 2-3),
   2-3h (Week 3-4), 1-2h (Week 4+ optional).

Q: What if something breaks during implementation?
A: Easy rollback: `git reset HEAD~1 && git checkout -- .` Patches are applied
   one at a time, making debugging simple.

Q: Which browsers need testing?
A: Chrome 143+ (target), Chrome 120-142 (fallbacks), Safari 17+ (where possible),
   Apple Silicon M-series hardware (for ProMotion 120Hz).

Q: Can we implement selectively?
A: Yes. Patches are independent except Patch 1 (design tokens) which Patches 7-9
   depend on. Suggested order: 1→5→6→2→3→4→7→8→9.
