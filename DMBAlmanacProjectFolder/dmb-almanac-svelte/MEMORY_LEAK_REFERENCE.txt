================================================================================
BEFORE & AFTER PATTERNS
================================================================================

PATTERN 1: Event Listeners

❌ BEFORE (Leaks):
window.addEventListener('resize', handleResize);
document.addEventListener('click', handleClick);

✓ AFTER (Fixed):
const controller = new AbortController();
window.addEventListener('resize', handleResize, { signal: controller.signal });
document.addEventListener('click', handleClick, { signal: controller.signal });

// In cleanup:
controller.abort();

---

PATTERN 2: Timers/Intervals

❌ BEFORE (Leaks):
const timerId = setInterval(doThing, 5000);

✓ AFTER (Fixed):
const timerId = setInterval(doThing, 5000);
// Later in cleanup:
clearInterval(timerId);

---

PATTERN 3: D3 Selections

❌ BEFORE (Leaks):
let selection = select(element);
selection.append('g').selectAll('circle')...
// selection never cleared

✓ AFTER (Fixed):
let selection = select(element);
selection.append('g').selectAll('circle')...
// In cleanup:
selection.selectAll('*').remove();
selection = undefined;

---

PATTERN 4: WASM Pending Calls

❌ BEFORE (Leaks):
private pendingCalls = new Map();  // Unbounded
if (result) {
  this.pendingCalls.delete(id);  // Only on success
  resolve(result);
}

✓ AFTER (Fixed):
private pendingCalls = new Map();
private readonly MAX_PENDING_CALLS = 100;

// Check size limit
if (this.pendingCalls.size >= this.MAX_PENDING_CALLS) {
  // Clean up oldest
}

// Always delete on finish
const cleanup = () => {
  this.pendingCalls.delete(id);
};
resolve(cleanup);
reject(cleanup);

---

PATTERN 5: Svelte Component Cleanup

❌ BEFORE (Leaks):
onMount(() => {
  window.addEventListener('resize', handleResize);
  // No cleanup!
});

✓ AFTER (Fixed):
onMount(() => {
  window.addEventListener('resize', handleResize);

  return () => {
    window.removeEventListener('resize', handleResize);
  };
});

---

PATTERN 6: Store Subscriptions

❌ BEFORE (Leaks):
$effect(() => {
  const unsubscribe = store.subscribe(value => {
    // update something
  });
  // Never unsubscribe!
});

✓ AFTER (Fixed):
$effect(() => {
  const unsubscribe = store.subscribe(value => {
    // update something
  });

  return () => {
    unsubscribe();  // Cleanup return
  };
});

CHROME DEVTOOLS WORKFLOW
================================================================================

Step 1: Take Initial Snapshot
  DevTools > Memory > Take Heap Snapshot
  Label it: "Initial state"

Step 2: Perform Suspect Action
  Navigate 10 times
  Open/close visualizations
  Perform WASM operations

Step 3: Take Second Snapshot
  DevTools > Memory > Take Heap Snapshot
  Label it: "After 10 navigations"

Step 4: Compare Snapshots
  Select comparison view
  Filter by "New" allocations
  Sort by size or count

Step 5: Investigate
  Look for:
    - Event listeners
    - Detached DOM nodes
    - Simulation objects (D3)
    - Function closures

Step 6: Monitor Cleanup
  Navigate away from page
  Force garbage collection (trash icon)
  Memory should drop significantly

If memory stays high:
  -> You likely have a leak in that component/service

================================================================================
LEAK SIGNATURES IN HEAP SNAPSHOTS
================================================================================

RUM Event Listeners:
  Look for: "visibilitychange", "beforeunload", "pagehide"
  Count: Should be 1-2, not 50+
  Size: Small but numerous

WASM Pending Calls:
  Look for: "PendingCall" objects, Promise resolvers
  Count: Should be 0 after operations complete
  Size: Large with argument data

D3 Simulation:
  Look for: "Simulation", "Node", "Link" (d3-force)
  Count: Should be 0 after visualization unmounts
  Size: Large with full node/link arrays

Navigation Listeners:
  Look for: "navigate" EventListener
  Count: Should be 1, not 10+
  Size: Small but numerous

Detached Nodes:
  Look for: "HTMLDivElement", "SVGSVGElement" (gray icons)
  Count: Should be <10
  Size: Varies

In development, use provided memory-monitor.ts:

// Start monitoring
import { memoryMonitor } from '$lib/utils/memory-monitor';

memoryMonitor.start({ interval: 5000 });

// Get report
const report = memoryMonitor.getReport();
console.log(report.formatReport());

// Check for leaks
const checkpoint = memoryMonitor.checkpoint();
// ... perform operations ...
const comparison = memoryMonitor.compareToCheckpoint(checkpoint);
console.log(`Growth: ${comparison.heapGrowthMB}MB in ${comparison.timeElapsedSeconds}s`);

// Detect leaks
import { detectMemoryLeak } from '$lib/utils/memory-monitor';

detectMemoryLeak('My Feature', () => {
  // operation code
}, { iterations: 10, expectedGrowthMB: 1 });

================================================================================
PREVENTION CHECKLIST
================================================================================

Before Committing Code:

Event Listeners:
  [ ] All addEventListener have matching removeEventListener
  [ ] Or use AbortController with signal parameter
  [ ] Cleanup happens in component unmount/destroy

Timers:
  [ ] All setInterval have clearInterval calls
  [ ] All setTimeout with cleanup use returned ID
  [ ] Cleanup happens in component unmount/destroy

D3/DOM:
  [ ] D3 selections explicitly cleared in cleanup
  [ ] Simulations stopped and references nulled
  [ ] DOM nodes removed with .remove()

WASM:
  [ ] Pending calls cleaned up on success/error
  [ ] Worker terminated on cleanup
  [ ] Pending calls map has size limit

Stores/Subscriptions:
  [ ] Unsubscribe functions called in cleanup
  [ ] $effect returns cleanup function
  [ ] Stores cleared on component unmount

RED FLAGS IN CODE REVIEW
================================================================================

LEAK: addEventListener without removeEventListener
LEAK: setInterval without clearInterval in cleanup
LEAK: setTimeout that never gets cleared
LEAK: D3 selections not nulled after remove()
LEAK: No cleanup return from onMount()
LEAK: Closures capturing large objects
LEAK: Unbounded Maps/Arrays without limits
LEAK: Window/document global listeners in components
LEAK: Event handlers with no unsubscribe
LEAK: WASM operations without cleanup
