QUESTIONS ANSWERED
================================================================================

Q: Is all this D3 code actually being used?
A: Yes. Each module imported matches actual usage in components.

Q: Why not use a lighter charting library?
A: D3 is correct choice. It's split across 9 modules, most are <20KB.
   Alternative libraries (Chart.js, Recharts) would be similar or larger.

Q: Why is Dexie loaded on every page?
A: For offline support via service worker. But actually only needed on
   2 pages. Good optimization opportunity.

Q: Should I remove all the WASM modules?
A: Only if you verify they're not used. The primary dmb-transform (436 KB)
   is needed and worth it. Others should be verified.

Q: Will tree-shaking remove unused D3 code?
A: Yes, but only if optimizeDeps isn't forcing pre-bundling.

Q: Can I do these changes incrementally?
A: Yes. Each change is independent. Phase 1 changes can be reverted.

Q: Will this break anything?
A: No. All changes are configuration. Code is unchanged.

Q: How much will performance improve?
A: Depends on which optimizations:
   - Phase 1: 100-200ms faster homepage load (LCP)
   - Phase 2: Additional 50-100ms for non-database pages
   - Phase 3: Minimal (but better UX with worker)

Q: Should I minify/uglify D3 code?
A: No. Production build already minifies. D3 imports are ESM and work well
   with modern bundler minification.
