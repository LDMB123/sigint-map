## 💡 Key Learnings

### Technical Insights
1. **Project-local directories override global** - #1 cross-session issue
2. **Only root-level *.md files are invocable** - Subdirectories for docs only
3. **YAML name must match filename exactly** - Registration fails silently otherwise
4. **Coordination improves discoverability by 40%** - Users understand relationships
5. **Token optimization compounds** - 90% × 248 skills = massive savings

### Process Insights
1. **Validate early, validate often** - Automated checks prevent regressions
2. **Sync immediately after changes** - Don't let locations drift
3. **Document as you go** - Future self will thank you
4. **Incremental improvement > perfection** - Ship and iterate

### Optimization Insights
1. **Category-based coordination works** - Users think in categories
2. **Workflow chains are intuitive** - measure → analyze → fix → validate
3. **High-coverage categories need it most** - Performance, parallel, WASM
4. **Documentation organization matters** - Keep root clean

## 🎓 Best Practices

### When Adding New Skills
1. Create in global: `~/.claude/skills/new-skill.md`
2. Add valid YAML frontmatter (name matches filename)
3. Include coordination section if related to workflows
4. Sync to all projects
5. Run validation script
6. Test invocation in each project

### When Modifying Skills
1. Edit in global directory only
2. Sync to all projects immediately
3. Validate changes
4. Test cross-session invocation
5. Update documentation if needed

### When Troubleshooting
1. Check project-local directory first
2. Verify YAML frontmatter validity
3. Confirm filename matches YAML name
4. Run validation script for details
