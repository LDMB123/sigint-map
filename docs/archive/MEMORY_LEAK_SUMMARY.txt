================================================================================
CRITICAL FINDINGS (FIX TODAY)
================================================================================

1. StorageQuotaMonitor - Missing setTimeout Cleanup
   Location: app/src/lib/components/pwa/StorageQuotaMonitor.svelte:161-165
   Issue: Auto-dismiss timeout on cache clear not tracked for cleanup
   Impact: 2-5 MB memory leak per cache clear operation
   Fix: Track timeout ID and clear in effect return
   Time to fix: 5 minutes

2. LazyVisualization - Missing Timeout Cleanup in Promise.race
   Location: app/src/lib/components/visualizations/LazyVisualization.svelte:135-141
   Issue: Timeout created in Promise constructor not canceled on component unmount
   Impact: 3-8 MB memory leak with repeated import failures
   Fix: Track timeoutId and clearTimeout on success/error
   Time to fix: 10 minutes

3. VirtualList - ResizeObserver Not Fully Disconnected
   Location: app/src/lib/components/ui/VirtualList.svelte:243-246
   Issue: Observer reassigned without explicit disconnect of previous instance
   Impact: Edge case memory creep when itemHeight function toggling occurs
   Fix: Explicitly null observer references after disconnect
   Time to fix: 2 minutes

================================================================================
MEDIUM-RISK PATTERNS (THIS WEEK)
================================================================================

4. InstallPrompt - 6 Concurrent $effect Blocks
   Location: app/src/lib/components/pwa/InstallPrompt.svelte:89-230
   Issue: Complex inter-dependent effects reduce debuggability
   Impact: Potential race conditions, harder to trace leaks
   Fix: Consolidate related effects into 3-4 logical groups
   Time to fix: 20 minutes

5. Dropdown - Event Listener Attached in onMount
   Location: app/src/lib/components/anchored/Dropdown.svelte:97-106
   Issue: Manual lifecycle management less reactive than $effect
   Impact: Potential listener attachment failure if binding timing wrong
   Fix: Replace onMount with $effect for automatic dependency tracking
   Time to fix: 5 minutes

TECHNICAL DETAILS
================================================================================

LEAK TYPE 1: Missing Timer Cleanup (2 instances)
- setInterval/setTimeout created but not tracked
- No clearInterval/clearTimeout in cleanup
- Callback persists in memory after component unmount
- FIX: Store ID and clear in $effect return or onDestroy

LEAK TYPE 2: Observer Not Fully Cleaned (1 instance)
- Observer created and referenced
- Observer reassigned without prior disconnect
- Previous observer stays in memory
- FIX: Explicitly call disconnect() and null reference

LEAK TYPE 3: Lifecycle Pattern Antipattern (2 instances)
- Using onMount for persistent listeners
- Less reactive to prop/state changes
- Potential timing issues with bindings
- FIX: Use $effect for automatic dependency tracking

PREVENTION STRATEGIES
================================================================================

CODE PATTERNS TO FOLLOW:

✅ Timer Cleanup:
   let id = setInterval(...);
   return () => clearInterval(id);

✅ Event Listener Cleanup:
   window.addEventListener('event', handler);
   return () => window.removeEventListener('event', handler);

✅ Observer Cleanup:
   const obs = new ResizeObserver(...);
   return () => obs.disconnect();

✅ Subscription Cleanup:
   const unsub = store.subscribe(...);
   return () => unsub();

✅ Fetch Cleanup:
   const ctrl = new AbortController();
   fetch(url, { signal: ctrl.signal });
   // cleanup: ctrl.abort();
