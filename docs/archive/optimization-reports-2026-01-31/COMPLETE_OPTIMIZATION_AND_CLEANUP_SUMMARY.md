## 🎓 Key Discoveries

### 1. Project-Local Override Behavior
**Discovery:** `.claude/skills/` in project folders OVERRIDES global `~/.claude/skills/`

**Impact:** Skills weren't available in some sessions because projects lacked local directories

**Solution:** Created `.claude/skills/` in all active projects with full synchronization

### 2. Three Distinct File Systems
**Skills:** Invocable commands in `.claude/skills/*.md`
- YAML frontmatter required
- Root level only
- Scanned at session start

**Agents:** Custom definitions in `.claude/agents/**/*.md`
- YAML frontmatter required
- Any depth in agents/
- Organized by subdirectory

**Documentation:** Supporting files in `.claude/skills/_docs/*.md`
- No frontmatter required
- Separate from skills
- Professional organization

### 3. Documents vs Projects Separation
**Documents:** User's actual documents (PDFs, images, archives)
**Projects:** Development work, code, configurations

**Critical:** Projects scattered in Documents cause confusion and clutter

**Solution:** Strict separation with migration of all projects to projects/ directory

### 4. Log File Organization Pattern
**Problem:** Log files accumulate in root and project directories

**Solution:** Create `_logs/` subdirectories for all generated logs

**Benefit:** Clean roots, organized logs, easy to find and clean up

### When Adding New Skills
1. Add to global: `~/.claude/skills/NEW_SKILL.md`
2. Sync to all project locations:
   ```bash
   rsync -av ~/.claude/skills/*.md ./.claude/skills/ --exclude='_*'
   rsync -av ~/.claude/skills/*.md projects/dmb-almanac/.claude/skills/ --exclude='_*'
   ```

### When Creating New Projects
