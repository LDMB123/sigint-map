## 🎓 Key Learnings

### Critical Discoveries
1. **Project-local override:** `.claude/skills/` in projects overrides global
2. **Three file systems:** Skills, Agents, Documentation - each with distinct purposes
3. **Coordination value:** Workflows that chain skills create compound value
4. **Metadata matters:** Professional descriptions improve discoverability

### Best Practices Established
1. Skills → `.claude/skills/*.md` (root only)
2. Docs → `.claude/skills/_docs/*.md` (subdirectory)
3. Agents → `.claude/agents/**/*.md` (any depth)
4. Sync all project locations when adding skills
5. Maintain 100% metadata quality

## 🔧 Maintenance Guide

### Monthly Health Check
```bash
# Run comprehensive validation
/tmp/final-comprehensive-check.sh

# Check skills quality
/tmp/final-skills-validation.sh

# Examine .md files
/tmp/final-md-examination-report.sh
```

### When Adding New Skills
1. Add to global: `~/.claude/skills/NEW_SKILL.md`
2. Sync to projects:
   ```bash
   rsync -av ~/.claude/skills/*.md ./.claude/skills/ --exclude='_*'
   rsync -av ~/.claude/skills/*.md projects/dmb-almanac/.claude/skills/ --exclude='_*'
   rsync -av ~/.claude/skills/*.md projects/emerson-violin-pwa/.claude/skills/ --exclude='_*'
   ```
3. Validate: `/tmp/final-skills-validation.sh`

### When Creating New Projects
1. Create skills directory:
   ```bash
   mkdir -p ./projects/NEW_PROJECT/.claude/skills
   ```
2. Sync from global:
   ```bash
   rsync -av ~/.claude/skills/*.md ./projects/NEW_PROJECT/.claude/skills/ --exclude='_*'
   ```
3. Validate sync count matches global
