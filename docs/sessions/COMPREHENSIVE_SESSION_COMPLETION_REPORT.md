## 🔍 Key Discoveries

### 1. Project-Local Override Behavior
**Discovery:** `.claude/skills/` in project folders OVERRIDES global `~/.claude/skills/`

**Impact:** Skills weren't available in emerson-violin-pwa sessions because project lacked local directory

**Solution:** Created `.claude/skills/` in all active projects with full sync

### 2. Three Distinct File Systems
**Skills:** Invocable commands (`.claude/skills/*.md`)
- Purpose: User-invocable skills for Claude Code
- Requirements: YAML frontmatter
- Location: Root level only

**Agents:** Custom definitions (`.claude/agents/**/*.md`)
- Purpose: Specialized agent definitions
- Requirements: YAML frontmatter
- Location: Any depth in agents/

**Documentation:** Supporting files (`.claude/skills/_docs/*.md`)
- Purpose: Guides, templates, examples
- Requirements: None
- Location: _docs/ subdirectory

### 3. Optimal Organization Pattern
**Best Practice:**
- Keep skills and docs separated
- Use _docs/ subdirectory for documentation
- Maintain perfect frontmatter across all skills
- Sync all project locations when adding skills
