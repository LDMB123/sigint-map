FUTURE PROJECTS

For new projects, choose strategy:

OPTION A: Use Global Skills (Recommended for most)
  - Don't create .claude/skills/
  - Let Claude Code use ~/.claude/skills/
  - Pros: Auto-sync, one source of truth
  - Cons: Can't customize per-project

OPTION B: Copy Skills to Project
  mkdir -p .claude/skills
  rsync -av ~/.claude/skills/*.md .claude/skills/
  - Pros: Project-specific customization
  - Cons: Must re-sync on skill updates

OPTION C: Symlink
  ln -s ~/.claude/skills .claude/skills
  - Pros: Auto-sync like Option A
  - Cons: Symlink might break
