PRIORITY 1: CRITICAL FIXES (Do immediately)
================================================================================

FIX #1: Border Width Animation (scrollBorderAnimate)
Location: /src/lib/motion/scroll-animations.css:454-461
Problem: border-width triggers layout recalc (60fps)
Fix: Replace with box-shadow inset (GPU, 120fps)

Current:
  @keyframes scrollBorderAnimate {
    from { border-width: 0; }
    to { border-width: 1px; }
  }

Replace with:
  @keyframes scrollBorderAnimate {
    from { box-shadow: inset 0 0 0 0px var(--color-primary-600); opacity: 0.7; }
    to { box-shadow: inset 0 0 0 1px var(--color-primary-600); opacity: 1; }
  }

Time: 5 minutes
Impact: +2% FPS on scroll animations


FIX #2: Remove scroll-position will-change
Location: /src/app.css:1172

Fix: Delete line

Remove:

FIX #4: Replace Clip-Path with Transform
Location: /src/lib/motion/scroll-animations.css:279-308
Problem: clip-path less efficient than transform on M-series
Fix: Replace clip-path with transform scaleX/scaleY

Current (scrollClipReveal):
  @keyframes scrollClipReveal {
    from { clip-path: inset(0 100% 0 0); }
    to { clip-path: inset(0 0 0 0); }
  }

Replace with:
  @keyframes scrollClipReveal {
    from {
      transform: scaleX(0);
      opacity: 0;
      transform-origin: left;
    }
    to {
      transform: scaleX(1);
      opacity: 1;
      transform-origin: left;
    }
  }

Time: 20 minutes
Impact: +5% FPS on M1 entry-level Macs

FIX #5: Add content-visibility to Lists
Location: Any long list pages (shows, songs, venues)
Problem: Off-screen items are rendered unnecessarily
Fix: Add content-visibility: auto with contain-intrinsic-size

Pattern:
  <style>
    .list-item {
      content-visibility: auto;
      contain-intrinsic-size: auto 120px;
    }
  </style>

Estimated heights:
  - Show items: 120px
  - Song items: 80px
  - Venue items: 150px

Time: 30 minutes
Impact: 20-40% faster initial render of long lists


FIX #6: Optimize Filter Blur Values
Location: /src/lib/motion/scroll-animations.css:509
Problem: Unnecessary blur intensity for M-series
Fix: Reduce from 10px to 8px

Current:
  filter: blur(10px);

Change to:
  filter: blur(8px);

Note: Blur guidelines for Apple Silicon
  - Light: 4px (no perf cost)
  - Medium: 8px (minimal cost) ← For animated
  - Heavy: 12px+ (only static)
  - Max safe animated: 8px
  - Max static: 20px

Time: 5 minutes
Impact: +2% FPS on blur animations
