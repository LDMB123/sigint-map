WHAT IS scheduler.yield()?

scheduler.yield() is a Chromium 129+ feature that:

  • Yields execution back to the browser event loop
  • Allows user input, rendering, animations to process
  • 50% faster than setTimeout(0)
  • On Apple Silicon: E-cores handle background during yield

Example:
  BEFORE: Process 250 items → yield → process 250 items (INP = 250ms)
  AFTER:  Process 50 items → yield → process 50 items (INP = 50ms)

APPLE SILICON OPTIMIZATION

This optimization maintains Apple Silicon efficiency:

  ✓ Respects Unified Memory Architecture (UMA)
  ✓ Smaller transactions still benefit from Metal GPU
