3. OFFLINE-FIRST ARCHITECTURE
================================================================================

A. IndexedDB Schema (src/lib/db/dexie/schema.ts - 712 lines)
   ✅ Version 3 (current) with optimized compound indexes
   ✅ Dexie.js 4.x with full TypeScript support
   ✅ 17 tables covering core entities and user data
   ✅ Denormalized structures for offline access
   ✅ Strategic compound indexes for common queries

   Core tables:
   • venues, songs, tours, shows, setlistEntries - Catalog data
   • guests, guestAppearances - Guest musicians
   • liberationList, songStatistics - Analytics
   • userAttendedShows, userFavoriteSongs, userFavoriteVenues - Local only
   • offlineMutationQueue - Failed mutation retry queue
   • curatedLists, releases, releaseTracks - Content management
   • syncMeta - Sync state tracking

B. Offline Mutation Queue (src/lib/services/offlineMutationQueue.ts - 900 lines)
   ✅ Production-grade offline handling with:

   Queue Design:
   • Stores to IndexedDB with id, url, method, body, status, retries, timestamps
   • FIFO ordering by createdAt
   • Status tracking: pending → retrying → completed/failed

   Retry Strategy:
   • Exponential backoff: 1s base, 2x multiplier per retry
   • Jitter: ±500ms to prevent thundering herd
   • Max 3 retries before marking failed
   • Non-retryable: 4xx errors (except 429 rate limit)
   • Retryable: 5xx, network errors, timeouts

   Processing:
   • Concurrent processing guard (prevents duplicate work)
   • 30s fetch timeout per mutation
   • Automatic retry scheduling at earliest retry time
   • Tracks results with success/failure/retrying status

   Public API:
   • queueMutation() - Add to queue
   • processQueue() - Process pending/retrying
   • getQueuedMutations() - Query all
   • getMutationsByStatus() - Filter by status
   • getQueueStats() - Aggregate metrics
   • clearCompletedMutations() - Cleanup
   • registerBackgroundSync() - Enable sync when closed
   • getStorageUsage() - Monitor quota

   Lifecycle:
   • initializeQueue() - Setup online/offline listeners
   • cleanupQueue() - Remove all listeners
   • Listens to: online, offline, visibilitychange events

❌ CRITICAL GAP: Background Sync Not Registered
   registerBackgroundSync() is exported but NEVER CALLED
   Impact: Offline mutations won't sync when app is closed
   Fix: Call in src/routes/+layout.svelte onMount()
   Effort: 1 line of code

⚠️ IMPORTANT: No Max Queue Size
   Queue could grow unbounded if mutations consistently fail
   Impact: Could consume entire storage quota
   Fix: Add MAX_QUEUE_SIZE = 500 constant and check before add
   Effort: 10 minutes
