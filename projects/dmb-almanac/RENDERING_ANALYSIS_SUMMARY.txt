DMB ALMANAC - CRITICAL RENDERING PATH ANALYSIS SUMMARY
======================================================

EXECUTIVE SUMMARY
================

The DMB Almanac app has well-architected data loading patterns but shows BLANK SCREEN for 250-350ms before displaying the loading screen. This is a critical UX issue caused by blocking onMount initialization.

CURRENT PERFORMANCE
===================

First Contentful Paint (FCP): 250-350ms BLANK SCREEN
  └─ Should be: 100-120ms with loading screen visible

Largest Contentful Paint (LCP): 3.5-8s (unchanged by fix)
  └─ Depends on data loading time

First Paint (FP): 250ms to white screen
  └─ Loading screen doesn't appear until 350ms

FEID (First Experience Input Delay): ~300ms
  └─ User clicks, sees blank for 300ms before feedback


ROOT CAUSE
==========

In /app/src/routes/+layout.svelte onMount() [lines 36-179]:

1. All initialization wrapped in Promise.allSettled([...])
2. dataStore.initialize() is ASYNC and AWAITED
3. This function runs isDataLoaded() IndexedDB query before returning
4. onMount BLOCKS until dataStore.initialize() completes (~250-350ms)
5. Only after onMount completes does $dataState update to 'loading'
6. React/Svelte re-render, loading screen finally appears

Timeline of current issue:
  100ms  - First Paint (browser wants to paint, but no content ready)
  150ms  - onMount fires (blocks rendering)
  250ms  - dataStore.initialize() still running (IndexedDB query)
  350ms  - First async task completes
  350ms  - status.set('loading') called
  400ms  - Re-render triggered
  400ms  - Loading screen finally appears (too late!)


SOLUTION
========

Move to NON-BLOCKING initialization pattern:

1. In dataStore.initialize(): Remove `async`, return immediately
   └─ Let data loading happen in background via .then() chain

2. In +layout.svelte onMount(): Set loading status SYNCHRONOUSLY
   └─ status.set('loading') at START of onMount, before any awaits

3. All background tasks run in Promise.allSettled() without blocking
   └─ SW registration, data loading, etc. all complete normally

Result: status.set('loading') → immediate re-render → loading screen visible

New timeline:
  100ms  - First Paint (loading screen visible!)
  150ms  - onMount fires
  160ms  - status.set('loading') called (synchronous, no await)
  170ms  - Promise.allSettled begins (non-blocking)
  200ms+ - Background tasks complete (SW, data, etc.)
