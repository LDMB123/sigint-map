================================================================================
BADGE BEHAVIOR
================================================================================

DISPLAY STATES:
  - 1 pending mutation: "1" badge
  - 5 pending/retrying: "5" badge
  - 0 mutations: (badge cleared)
  - 99+ mutations: "99+" (overflow)

REAL-TIME UPDATES:
  User offline, adds 3 mutations:
    Step 1: Add favorite → Badge: "1"
    Step 2: Add to playlist → Badge: "2"
    Step 3: Create note → Badge: "3"
  
  User comes online:
    Processing: 3 → 2 → 1 → (clear)
    User sees real-time sync progress

PERSISTENCE:
  - Badge remains visible when app closed
  - Badge visible in app drawer
  - Badge visible in recent apps
  - Motivates user to sync when online

BADGE COUNTING LOGIC:

  // Only counts active mutations
