## Category 1: WASM Module Type Mismatches (CRITICAL)

### Root Cause
The `WasmExports` interface in `/app/src/lib/wasm/types.ts` defines functions with JavaScript naming conventions (camelCase), but the actual WASM modules generated by `wasm-bindgen` export functions with Rust naming conventions (snake_case).

### Affected Errors (35 total)

#### queries.ts & search.ts (14 errors)
```
TS2345: Argument of type '"global_search"' is not assignable to parameter of type 'keyof WasmExports'
TS2345: Argument of type '"get_tour_stats_by_year"' is not assignable...
TS2345: Argument of type '"count_openers_by_year"' is not assignable...
```

**Pattern:** All query functions use snake_case names from actual WASM, but `WasmExports` only defines camelCase.

#### worker.ts (4 errors)
```
TS2367: This comparison appears to be unintentional because the types 'keyof WasmExports' and '"build_search_index"' have no overlap
TS2551: Property 'free_search_index' does not exist on type 'WasmExports'. Did you mean 'freeSearchIndex'?
```

**Pattern:** Worker code checks for snake_case function names that don't exist in type definition.

#### bridge.ts & advanced-modules.ts (17 errors)
```
TS2352: Conversion of type 'typeof import(".../dmb_transform")' to type 'WasmExports' may be a mistake...
  Property 'SetlistPredictor' is missing...
  Property 'daysBetween' is missing...
  Property 'init' is missing...
```

**Pattern:** Type assertions assume WASM modules have properties they don't export.

### Solution

**Option A: Extend WasmExports with snake_case aliases**
```typescript
export interface WasmExports {
  // Existing camelCase definitions...
  globalSearch(...): string;

  // Add snake_case aliases for actual WASM exports

  get_tour_stats_by_year(...): any;
  count_openers_by_year(...): any;
  count_closers_by_year(...): any;
  count_encores_by_year(...): any;
  get_show_ids_for_song(...): any;
  get_show_ids_for_guest(...): any;
  get_year_breakdown_for_guest(...): any;
  get_year_breakdown_for_song(...): any;
  get_year_breakdown_for_venue(...): any;
  get_top_songs_by_performances(...): any;
  aggregate_shows_by_year(...): any;
  get_tours_grouped_by_decade(...): any;

  // Search index functions
  build_search_index(entities_json: string): number;
  free_search_index(index_handle: number): void;
}
```

**Option B: Use WASM module type directly (RECOMMENDED)**
```typescript
// Import actual WASM type definitions
import type * as DmbTransform from '../../../wasm/dmb-transform/pkg/dmb_transform';

// Create union type of all WASM modules
```
