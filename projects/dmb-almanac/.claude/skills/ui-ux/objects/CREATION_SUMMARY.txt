## Anti-Pattern Coverage (32 Total)

Structured as "BAD (example)" + "Why it's bad" + "GOOD (solution)"

**icon-perfection.md** (5 anti-patterns):
1. Icon fonts vs SVG
2. Blurry icons from scaling
3. Misaligned icons in buttons
4. Inconsistent stroke widths
5. Icons without accessibility

**button-craft.md** (6 anti-patterns):
1. Ambiguous button labels
2. Hidden or unclear focus states
3. Touch target sizing extremes
4. Loading state not disabling button
5. Inconsistent button styles
6. Icons without proper spacing

**input-excellence.md** (6 anti-patterns):
1. Placeholder-only labels
2. Removed focus styles
3. Font size < 16px on mobile
4. Generic error messages
5. Missing aria-describedby
6. Labels not associated with inputs

**card-architecture.md** (5 anti-patterns):
1. Inconsistent card padding
2. Content overflow without truncation
3. Images breaking aspect ratio
4. No hover state feedback
5. Fixed heights causing overflow

**image-presentation.md** (5 anti-patterns):
1. Same image for all devices
2. Loading all images eagerly
3. Layout shift from missing aspect-ratio
4. No alt text or placeholder
5. Modern formats without fallback

**divider-spacing.md** (5 anti-patterns):
1. Arbitrary spacing values
2. Cramped layout (no whitespace)
3. Using margin for flex spacing
4. Inconsistent divider usage
5. Overusing dividers

## Performance Notes

### Image Optimization
- Lazy loading reduces initial page load
- WebP/AVIF reduces bandwidth by 30-50%
- aspect-ratio prevents layout shift (improves CLS)
- LQIP provides perceived speed

### CSS Optimization
- CSS custom properties enable dynamic theming
- Grid/Flexbox more performant than floats
- Will-change hints GPU acceleration
- Reduced repaints with transform

### JavaScript Optimization
- Debounced validation (500ms)
- Intersection Observer for lazy load fallback
- No animation flicker with appropriate timing
- Memory-efficient component patterns
