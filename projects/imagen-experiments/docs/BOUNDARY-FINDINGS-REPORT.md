## Filter Escalation System

### IMAGE_SAFETY (Tier 1 - Recoverable)

**Characteristics:**
- Single boundary violation
- Typically recoverable by adjusting 1 element
- Can retry with modifications

**Examples:**
- Fishnet hosiery in isolation
- Deep V plunging neckline alone
- <500 word prompts
- >1100 word ultra-dense prompts
- Gaze duration >9 seconds

**Recovery strategy:** Adjust single violating element, retry with 700-900 word shield.

### IMAGE_PROHIBITED_CONTENT (Tier 2 - Severe)

**Characteristics:**
- Multiple boundary violations combined
- Requires significant adjustment (remove 2+ elements)
- Stronger filter response

**Examples:**
- Two-piece + late timing (2:30am+) + seductress language
- Wrap dress "significantly loosened maximum" + 44cm "minimal length" + late timing
- "Last call" timing + other boundaries

**Recovery strategy:** Remove 2+ boundary elements, use conservative alternatives, 700-900 word shield.

## Limitations & Future Work

### Known Limitations

1. **Single model:** Boundaries specific to Imagen 3 Pro, don't transfer to DALL-E/Midjourney/SD
2. **Temporal validity:** Google may update filters invalidating discoveries
3. **Context-specific:** Nashville honky-tonk setting; other contexts need separate mapping
4. **Sample size:** 24 generations insufficient for statistical significance (need 100+)
5. **Reference image dependency:** All tests used consistent facial reference; different faces may behave differently

### Recommended Future Research

**High Priority:**

1. **Minimum viable shield:** Binary search for exact word count threshold (currently ~700)
2. **Component ablation:** Remove shield components individually to find essential vs redundant
3. **Temporal stability:** Re-test boundaries monthly to detect filter updates
4. **Cross-context validation:** Test boundaries in non-Nashville settings (beach, studio, urban)

**Medium Priority:**

5. **Cross-model comparison:** Map boundaries for DALL-E 3, Midjourney v6, SD3
